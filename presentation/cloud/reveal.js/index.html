Introduction

1. Introduction à Alibaba Cloud

Contexte

Le cloud computing a profondément transformé les technologies de l'information modernes. De nombreuses entreprises migrent leurs centres de données locaux vers le cloud pour réduire les coûts et améliorer leur agilité opérationnelle.

Pourquoi ce choix

Plutôt que de construire et maintenir de nouveaux datacenters coûteux, les entreprises préfèrent s'appuyer sur des services cloud prêts à l'emploi et évolutifs.

Réduction des coûts Agilité Scalabilité Déploiement rapide

Alibaba Cloud : Position et contexte

Position dans l'industrie

Selon les recherches de Gartner, Alibaba Cloud conserve une position solide parmi les fournisseurs cloud mondiaux. Fondee en 2009 pour soutenir les plateformes du groupe Alibaba, elle sert aujourd'hui des organisations dans le monde entier.

Signal de croissance

Alibaba Cloud a enregistre une croissance positive des profits apres 11 ans, acceleree par la vague de numerisation liee au COVID-19 et l'adoption mondiale du cloud.

Services Alibaba Cloud

Modeles de service principaux

  • IaaS : Infrastructure elastique et calcul virtualise.
  • PaaS : Services de plateforme geres pour le deploiement applicatif.
  • Couche de calcul : Machines virtuelles ECS sur hyperviseurs Xen et KVM.

Services de l'ecosysteme

  • Service de stockage d'objets
  • CDN (reseau de diffusion de contenu)
  • Cloud Container Service base sur Docker
  • Cloud prive Apsara Stack
  • Famille Apsara DB
  • Cloud Intelligence Brain (plateforme IA)

Cout et performance

Avantage economique et compromis

Atout cout

Alibaba Cloud peut etre environ 25% moins cher que d'autres grands fournisseurs cloud.

Consideration regionale

Le service peut afficher des temps de reponse plus lents dans des pays comme les Etats-Unis, tout en restant efficace pour les charges lourdes dans ses regions fortes.

Synthese

Resume pour les decideurs

  • Alibaba Cloud est un leader regional en Chine et en Asie-Pacifique sur les services d'infrastructure.
  • Son portefeuille couvre le calcul, le stockage, le reseau, les bases de donnees, les conteneurs, le cloud prive et l'IA.
  • La competitivite en cout est forte, surtout pour les charges applicatives gourmandes en ressources.
  • Les choix d'architecture doivent tenir compte des compromis de latence selon la geographie.
